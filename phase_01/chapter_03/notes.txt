## **Topic 3.1: Classes & Objects**

### ğŸ”¹ What is OOP?

Object-Oriented Programming is a way of structuring programs by **grouping data (attributes) and behavior (methods)** together into **objects**.

Real-world analogy:

* **Class** â†’ Blueprint (Car)
* **Object** â†’ Actual thing (Your car)

---

### ğŸ”¹ Defining a Class

```python

    pass
```

* `class` keyword defines a class
* Class names follow **PascalCase**

---

### ğŸ”¹ The `__init__` Method (Constructor)

```python
class Person:
    def __init__(self, name, age):
        self.name = name
        self.age = age
```

* Automatically runs when an object is created
* `self` refers to the **current object**
* Used to initialize **instance attributes**

---

### ğŸ”¹ Creating Objects (Instances)

```python

p2 = Person("Bob", 30)
```

Each object has **its own data**.

---

### ğŸ”¹ Instance Attributes vs Class Attributes

#### Instance Attribute (unique per object)

```python

self.age
```

#### Class Attribute (shared by all objects)

```python

    species = "Human"
```

Access:

```python
Person.species
p1.species
```

---

### ğŸ”¹ Instance Methods

```python

    def greet(self):
        return f"Hello, my name is {self.name}"
```

* Must take `self` as first parameter
* Operates on object data

## ğŸ” Encapsulation

### ğŸ”¹ What is Encapsulation?

Encapsulation means **bundling data and methods together** and **restricting direct access** to internal details.

Python uses **convention**, not strict enforcement.

---

### ğŸ”¹ Private Attributes (by convention)

```python

    def __init__(self, balance):
        self._balance = balance
```

* `_balance` means: â€œDonâ€™t access directlyâ€
* Still accessible, but discouraged

---

### ğŸ”¹ Controlled Access (Best Practice)

```python

    def __init__(self, balance):
        self._balance = balance

    def get_balance(self):
        return self._balance
```

## ğŸ§¬ Inheritance

### ğŸ”¹ What is Inheritance?

A child class **inherits properties and methods** from a parent class.

---

### ğŸ”¹ Basic Inheritance

```python

    def speak(self):
        print("Animal speaks")

class Dog(Animal):
    pass
```

```python
d = Dog()
d.speak()
```

---

### ğŸ”¹ Using `super()`

```python
class Dog(Animal):
    def __init__(self):
        super().__init__()
```

* Calls the parentâ€™s method
* Avoids hardcoding parent class name

---

### ğŸ”¹ Method Overriding

```python

    def speak(self):
        print("Dog barks")
```

Child method **replaces** parent method.

## ğŸ”„ Polymorphism

### ğŸ”¹ What is Polymorphism?

Same method name, **different behavior**, depending on the object.

---

### ğŸ”¹ Duck Typing (Python-style)

```python

    def fly(self):
        print("Flying")

class Airplane:
    def fly(self):
        print("Flying with engines")
```

```python

    obj.fly()
```

Python doesnâ€™t care about type â€” only behavior.

---

### ğŸ”¹ Polymorphism via Inheritance

```python

    def area(self):
        pass

    def area(self):
        return 4 * 4
```

## ğŸ·ï¸ Class Methods

## ğŸª„ Property Decorators

### ğŸ”¹ `@property` (getter)

Access like an attribute:

```python

```

---

### ğŸ”¹ `@setter`

```python

```
