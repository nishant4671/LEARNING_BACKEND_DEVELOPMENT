# 2.1 FUNCTIONS (The Core Weapon of Every Programmer)

## âœ… 2.1.1 Why Functions Exist (Industry View)
In real projects, code is big and changing daily.
Functions help you:
- âœ… **Reuse logic** (write once, use 100 times)
- âœ… **Reduce bugs** (one fix updates everywhere)
- âœ… **Increase clarity** (readable + organized)
- âœ… **Make testing possible**
- âœ… **Make collaboration possible**

### Bad Code (repeating logic)
```python
a = 10
b = 5
print(a + b)

x = 20
y = 30
print(x + y)
````

### Good Code (reusable logic)

```python
def add(n1, n2):
    return n1 + n2

print(add(10, 5))
print(add(20, 30))
```

## âœ… 2.1.2 Defining a Function (`def`)

Syntax:

```python
def function_name(parameters):
    # code
    return value
```

Example:

```python

    print("Hello from Python!")
```

Calling it:

```python

```

âœ… Key things:

* `def` declares a function
* indentation matters
* function name should follow `snake_case`

---

## âœ… 2.1.3 Parameters vs Arguments (VERY IMPORTANT)

### Parameter = variable inside function definition

### Argument = actual value passed during call

```python
def add(a, b):        # a, b are PARAMETERS
    return a + b

print(add(10, 20))    # 10, 20 are ARGUMENTS
```

---

## âœ… 2.1.4 Return Values (`return`) vs Printing

### `print()` shows output on screen

### `return` sends output back to caller

Example:

```python
def square(n):
    return n * n

result = square(5)
print(result)
```

âœ… Why return matters in backend:
Because APIs need to compute values and pass them forward.

### If you forget return

```python
def square(n):
    n * n

print(square(5))  # None
```

Because function ends without returning â†’ automatically returns `None`.

## âœ… 2.1.5 Default Arguments (Production Pattern)

```python
def greet(name="Guest"):
    print("Hello", name)

greet("Nishant")
greet()
```

âœ… Default args make functions flexible.

âš ï¸ WARNING: Mutable default arguments are dangerous:

```python
def add_item(item, items=[]):  # BAD PRACTICE
    items.append(item)
    return items
```

Because the list is created only once (shared across calls).

âœ… Correct way:

```python
def add_item(item, items=None):
    if items is None:
        items = []
    items.append(item)
    return items
```

## âœ… 2.1.8 Function Scope (Local vs Global)

### Local scope

Variables inside functions stay inside:

```python
def test():
    x = 10
    print(x)

test()
# print(x)  # NameError
```

### Global scope

Variables outside functions:

```python
x = 100

def show():
    print(x)

show()
```

âœ… A function can read global variable.

âš ï¸ But to MODIFY global variable:

```python
x = 100

def change():
    global x
    x = 200

change()
print(x)
```

âœ… In industry: using `global` is usually discouraged.
Better: return value and update.

## âœ… 2.1.13 map() and filter() (Functional Tools)

### `map(function, iterable)` â†’ transforms each element

Example:

```python
nums = [1, 2, 3, 4]
squares = list(map(lambda x: x*x, nums))
print(squares)
```

Same using loop:

```python
squares = []
for n in nums:
    squares.append(n*n)
```

âœ… `map()` is compact and clean for transformation.

---

### `filter(function, iterable)` â†’ keeps only True values

Example:

```python
nums = [1, 2, 3, 4, 5, 6]
evens = list(filter(lambda x: x % 2 == 0, nums))
print(evens)
```

âœ… filter() = selection tool.

## âœ… 2.1.14 Common Industry Patterns with Functions

### Input validation function

```python

    return isinstance(age, int) and age > 0

print(is_valid_age(19))
print(is_valid_age(-5))
print(is_valid_age("19"))
```

### Pure function vs impure function

âœ… Pure function: no side effects

```python

    return a + b
```

âŒ Impure function: depends on external state

```python

    global total
    total += x
```

âœ… Industry prefers pure functions when possible.

# 2.2 MODULES & PACKAGES (How Real Python Projects Work)

## âœ… 2.2.1 What is a Module?

A module is simply a **Python file**.

Example:

* `math_utils.py` âœ… is a module
* `main.py` âœ… is a module

If you write:

```python
# math_utils.py
def add(a, b):
    return a + b
```

You can use it in another file:

```python
# main.py
import math_utils

print(math_utils.add(2, 3))
```

---

## âœ… 2.2.2 What is a Package?

A package is a **folder** containing Python modules.

Example:

```
project/
  main.py
  utils/
    __init__.py
    helpers.py
```

âœ… The folder becomes a package when it has `__init__.py`.

## âœ… 2.2.7 `__name__ == "__main__"` (VERY IMPORTANT)

This decides if file is run directly or imported.

Example:

```python

    return a + b

    print(add(10, 20))
```

âœ… If you run:

```bash

```

then `__name__ == "__main__"` â†’ True

âœ… If imported:

```python

```

then that block wonâ€™t run.

Industry use:

* prevents accidental code execution during import
* makes modules usable as scripts too
